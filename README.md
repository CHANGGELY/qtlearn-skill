# A股量化培训大师Skill

一个面向零基础到进阶用户的 A 股量化学习与实战训练 Skill。

## 一、项目定位
A股量化培训大师Skill 不是普通问答工具，而是一个“会诊断、会教学、会测评、会路由、会项目化训练”的量化学习总 Skill。

它的目标不是只回答一个问题，而是把用户逐步培养成具备以下能力的人：
- 能理解 A 股量化的核心概念
- 能把模糊炒股想法变成量化策略
- 能获取和处理历史数据
- 能设计并解释回测
- 能理解风险控制与组合管理
- 能具备走向实盘前的工程认知

## 二、适合谁用
- 想系统学习 A 股量化的小白
- 已经会一点 Python，但不会系统做策略的人
- 有炒股经验，但不会把主观逻辑量化的人
- 想做量化项目、作品集、比赛展示的人

## 三、如何调用
### 1. 直接对话
你可以直接说你的问题，例如：
- “我完全不懂量化，从哪开始？”
- “帮我把这个选股想法变成策略”
- “为什么我的回测很好但实盘不行？”

### 2. 输入命令触发
你也可以用命令快速进入不同模式：
- `/qtlearn`：开始学习诊断
- `/qtpath`：生成个性化学习路径
- `/qtcheck`：测评、批改、检查理解
- `/qtstrategy`：进入策略设计专项
- `/qtbacktest`：进入回测专项
- `/qtrisk`：进入风控专项
- `/qtlive`：进入实盘专项
- `/qtproject`：启动项目制学习
- `/qtskill`：查看可用子 skill
- `/qtreview`：进入复盘与诊断
- `/qtfaq`：快速问答

### 3. 推荐起手式
最推荐的输入是：
- `/qtlearn 我想系统学A股量化，但我现在是零基础`

这样 Skill 会先判断你的基础、目标和时间投入，再给你最短学习路径。

## 四、操作步骤
1. 先说出你的当前水平或目标
2. Skill 自动判断你适合学什么
3. 它会把你路由到对应子 skill
4. 你按它给的小任务继续回答或实操
5. 它再检查、纠错，并给出下一步

## 五、最小 demo 流程
### Demo 1：零基础学习
用户输入：
“/qtlearn 我想学A股量化，但我现在是零基础，平时每周只有3小时。”

Skill 回复路径：
- 先问基础问题
- 判断用户阶段
- 给出最短学习路线
- 路由到 onboarding / basics

### Demo 2：策略想法转量化
用户输入：
“我觉得低估值股票更稳，帮我变成策略。”

Skill 回复路径：
- 拆解模糊观点
- 定义可测条件
- 输出策略草案
- 路由到 strategy_design / backtest

### Demo 3：回测诊断
用户输入：
“为什么我回测很好但实盘不行？”

Skill 回复路径：
- 排查未来函数、过拟合和成本
- 提醒样本外验证
- 给出风控和执行建议
- 路由到 backtest / risk_control / live_trading

## 六、失败示例（更像教练）
### 失败示例 1：用户问题太泛
用户：我想学量化。

Skill 应该追问：
- 你现在会 Python 吗？
- 你想先学概念还是实战？
- 你偏向选股、择时还是实盘？

### 失败示例 2：用户想直接下结论
用户：你直接告诉我什么股票会涨。

Skill 应该回应：
- 不能直接给确定性投资建议
- 但可以把你的选股逻辑改写成可回测的研究问题

### 失败示例 3：用户目标和时间不匹配
用户：我每周 3 小时，但想一周做完实盘系统。

Skill 应该说明：
- 目标需要拆阶段
- 先完成策略、回测和风控，再谈实盘

## 七、核心设计亮点
### 1. 总 Skill + 子 Skill 结构
总 Skill 负责诊断、编排、路由和追踪学习路径。
子 Skill 负责具体模块教学与任务执行。

### 2. 主动命令 + 被动语义触发
用户既可以输入命令，也可以直接自然对话。

### 3. 苏格拉底式教学
不是一口气灌输知识，而是：
- 先提问了解用户水平
- 再讲用户最可能感兴趣的部分
- 再布置一个最小练习
- 再检查理解并进阶

### 4. 项目制闭环
整个 Skill 强调闭环：
诊断 → 教学 → 练习 → 测评 → 纠错 → 项目 → 复盘

## 十、与现有量化项目的结合
这个 Skill 可以直接接入你现有的量化知识、策略和因子库：
- 在 **策略设计** 阶段调用你已有的选股逻辑
- 在 **数据** 阶段复用你现成的数据处理经验
- 在 **回测** 阶段接入你已有的回测框架和指标口径
- 在 **风控** 阶段结合你现有仓位管理、风险标签和暴露控制思路
- 在 **项目制训练** 阶段用你真实的策略案例做教学样本

这样它不是空泛教学，而是能和你的真实项目互相喂养。

## 十一、参赛展示建议
如果用于比赛展示，建议突出以下三点：

### 1. 它不是“回答量化问题”
而是“把用户培养成量化研究员”。

### 2. 它的核心价值是个性化学习路径
不同用户会被诊断到不同阶段，然后进入不同子 Skill。

### 3. 它兼顾知识、策略、工程和合规
不仅会讲概念，也会讲回测、风控、数据和实盘约束。
